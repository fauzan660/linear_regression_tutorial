How does the computer decide which line is better?

You’ve already drawn many lines by changing the values of m and c. Some lines feel close to the points, others clearly miss.

As humans, we can judge this by looking. A computer can’t. It needs a number that tells it how bad a line is.

That number comes from error.

For a single point, error is simply the difference between the real value and what the line predicts.

\[ \text{error} = y - \hat{y} \]

Points above the line give positive error, points below give negative error. But the sign itself is not important.

What matters is how far the point is from the line.

One simple way to measure this is to take the absolute value of each error and average them.

\[ \text{MAE} = \frac{1}{n} \sum |y - \hat{y}| \]

This tells us, on average, how far the line is from the points. Every mistake is treated equally.

A small miss and a big miss both increase the error in a straight line.

Another way is to square each error before averaging.

\[ \text{MSE} = \frac{1}{n} \sum (y - \hat{y})^2 \]

Squaring changes things.

Small errors stay small, but large errors grow very fast. This means points that are far from the line matter much more.

This difference becomes important when the computer tries to improve the line.

With absolute error, all points push the line with the same strength. With squared error, badly predicted points push much harder.

Because of this, squared error gives clearer guidance when adjusting m and c.

So two lines that look almost the same to us can be very different to the computer.

One might have small, evenly spread mistakes. Another might have one or two very bad ones.

Squared error makes sure those big mistakes cannot be ignored.

In short:

Absolute error asks:
“How far am I, on average?”

Squared error asks:
“Where am I messing up the most?”

That is why most learning algorithms prefer squared error when deciding how to move the line.
